📚 Visão Geral

O objetivo deste projeto é desenvolver uma plataforma educacional baseada em metodologias ativas utilizando arquitetura backend moderna.

Cada grupo desenvolverá um módulo independente dentro de um único ecossistema.

🔄 Provider Pattern

O Provider Pattern permite trocar a estratégia de acesso aos dados sem alterar o restante do sistema.

📦 Interface Base

class DatabaseProvider:

    def get_all(self):
        pass

    def create(self, data):
        pass

🏭 Provider Factory

provider = ProviderFactory.create("sql")
O restante do sistema não precisa saber se os dados vêm de SQL, Supabase ou FastAPI.
