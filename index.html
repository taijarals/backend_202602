📘 Projeto Final

📌 Visão Geral

Neste projeto, os alunos deverão desenvolver uma simulação de ecossistema em linguagem C utilizando estruturas de dados e algoritmos para controlar entidades, movimentações, interações e evolução do ambiente ao longo do tempo.

O sistema deverá executar automaticamente diversas rodadas até que uma condição de vitória seja atingida.

🔍 Focos principais do projeto

  • Estruturas de dados
  • Organização lógica
  • Escalabilidade
  • Eficiência computacional
  • Uso de memória
  • Simulação dinâmica

🎯 Objetivos Pedagógicos

  • Aplicar estruturas de dados em problemas reais
  • Trabalhar com matrizes, vetores, structs e ponteiros
  • Manipular memória em C
  • Implementar algoritmos de simulação
  • Refletir sobre eficiência computacional
  • Compreender escalabilidade de algoritmos
  • Analisar impacto de decisões arquiteturais

🎨 Tema e Personalização

🧩 Tema Base

O tema padrão da simulação será:

  • Ovelha
  • Lobo
  • Caçador

🎭 Personalização Permitida

Os personagens poderão ser alterados livremente.

  • Humanos, zumbis e soldados
  • Peixes, tubarões e mergulhadores
  • Robôs, vírus e antivírus
  • Magos, monstros e cavaleiros
  • Aliens, astronautas e caçadores

⚠️ Restrição da Personalização

Mesmo alterando os personagens, o grupo deverá manter:

  • A mesma complexidade lógica
  • Três facções principais
  • Interações equivalentes
  • Reprodução/evolução
  • Combate/sobrevivência
  • Obstáculos fixos no mapa

🌍 Tabuleiro

📏 Dimensão

O tamanho do tabuleiro deverá ser escolhido pelo usuário no início da execução.

O menor tamanho permitido será 5x5.

O sistema deverá funcionar corretamente para diferentes tamanhos.

📊 Proporções Automáticas

A quantidade de personagens e obstáculos deverá ser calculada automaticamente com base no tamanho do tabuleiro.

Elemento Percentual
Facção 1 12%
Facção 2 12%
Facção 3 5%
Obstáculos 10%

📌 Regras

  • Todos os elementos devem ser posicionados aleatoriamente
  • Não pode haver sobreposição
  • Obstáculos não podem se mover
  • Personagens não podem nascer sobre obstáculos

🧭 Movimentação

  • Cima
  • Baixo
  • Esquerda
  • Direita

📌 Regras

  • A movimentação deverá ser aleatória
  • Não pode sair do mapa
  • Não pode atravessar obstáculos

⚔️ Regras de Interação

🔴 Relação de Predação/Combate

Uma facção deverá ser capaz de eliminar outra.

🟢 Relação de Proteção/Salvamento

Uma facção deverá proteger outra.

🟡 Relação de Risco

Uma facção deverá possuir condição de vulnerabilidade.

🔵 Reprodução/Evolução

Entidades iguais deverão poder gerar novas entidades.

🧱 Obstáculos

O sistema deverá possuir pelo menos dois tipos diferentes de obstáculos.

  • Árvore
  • Pedra
  • Casa
  • Rio
  • Muro

📌 Regras

  • Não se movem
  • Bloqueiam movimentação
  • Ocupam espaço permanentemente

🧠 Estruturas de Dados

Os grupos possuem liberdade para escolher qualquer estrutura de dados.

  • Matrizes
  • Vetores
  • Listas encadeadas
  • Pilhas
  • Filas
  • Structs
  • Ponteiros
  • Alocação dinâmica

⚠️ Importante

Não existe estrutura “correta”.

Parte da avaliação será baseada:

  • Na justificativa técnica
  • Na eficiência
  • Na organização da solução

📊 Análise de Memória

Após a entrega da primeira versão, o código será submetido a um analisador estático desenvolvido pelo professor.

🧠 O que será analisado

  • Uso de memória
  • Escalabilidade
  • Crescimento computacional
  • Impacto dos loops
  • Impacto das estruturas utilizadas

📌 O analisador considerará

  • Variáveis primitivas
  • Arrays
  • Structs
  • Alocação dinâmica
  • Quantidade de loops
  • Crescimento de memória por repetição

🚀 Desafios Extras

  • Inteligência artificial simples
  • Sistema de energia/fome
  • Clima
  • Múltiplos mapas
  • Estatísticas em tempo real
  • Eventos aleatórios
  • Diferentes biomas

📅 Considerações Finais

O objetivo do projeto não é apenas “fazer funcionar”.

O principal objetivo é compreender:

  • Como estruturas de dados impactam desempenho
  • Como algoritmos escalam
  • Como decisões arquiteturais afetam memória
  • Como diferentes soluções resolvem o mesmo problema com custos computacionais distintos
